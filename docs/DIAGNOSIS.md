### 已驗證的系統性缺陷（按嚴重度排序）

#### D-1【嚴重】回測存在時間混淆（time confound），分數桶不可比

現行 `backtest_signals.py` 把不同時間的樣本混在同一分數桶比較。
量化分數含「近期性」權重 → 高分樣本天生集中在最近幾週 → 高分桶
的 -9.6% 可能只反映「最近剛好回檔」，而非分數本身的性質。
**桶與桶不在同一段市場環境，任何跨桶結論都不可信。**

#### D-2【嚴重】沒有樣本外（out-of-sample）驗證

所有回測都在「事後全知」的資料上跑。規則（RSI<65、score≥70、
1.5×ATR 停損）從未在「先凍結規則 → 再看未來」的框架下驗證過。
沒有樣本外測試 = 無法宣稱任何預測力。

#### D-3【高】評分權重從未校準，且證據指向 recency 權重有害

回測已顯示高分桶（recency 高權重的產物）表現最差。八個評分
成分（frequency/recency/persistence/engagement/conviction/
theme_fit/catalyst/risk_penalty）的權重全是拍腦袋設的，
從未做過任何敏感度掃描。

#### D-4【高】訊號閾值全部未驗證

- BUY_WATCH 的 `score>=70`：70 这个數字無任何實證依據
- `RSI<65`、`RSI<40 出場`、`1.5×ATR 停損`、`R:R 1:2`：教科書
  預設值，未對本資料集驗證
- StockTwits `>55% bullish`、`<30% 觸發出場`：我上輪自設的
  ad-hoc 閾值，同樣未驗證

#### D-5【高】缺「經理人視角」的綜合判斷層

現有輸出是零散的：量化分數、技術訊號、情緒、評分卡各自獨立。
沒有一個把**所有真實證據**整合成單一結構化投資論述
（thesis → 證據 → 風險 → 結論 + 信心度）的層。使用者要自己
腦內合成，等於把最難的一步留給人。

#### D-6【中】position sizing 以 EMA20 為進場錨，脫離現價

`position_sizing(entry=EMA20)` 在價格遠離 EMA20 時給出
不可執行的進場區（如 AAOI 現價 $121，進場區卻在 $145-166）。
應以現價為錨、EMA20 為參考。

#### D-7【中】訊號無歷史記錄，無法追蹤命中率

`/api/signal` 每次即時計算，不落地。無法回答「上週的
EXIT_ALERT 後來對了嗎？」——沒有訊號歷史就永遠無法累積
真實命中率。

#### D-8【低】回測忽略交易成本與滑價

中位數 +5% 的甜蜜區在扣除點差/滑價後可能縮水；目前完全未建模。

## 第二部分：修改方案（等待執行）

### R-1 30 天樣本外驗證框架（對應 D-1, D-2）★本輪核心

**方法：時間切割（temporal holdout）**

```
時間軸:  [======= 訓練期 =======][== 最近 30 天：驗證期 ==]
                              T-30                    今天
```

1. 凍結截止日 = 最新價格日 − 30 個日曆日
2. 只用截止日**之前**的資料計算每檔股票的量化分數、技術指標、
   訊號狀態（signals.py 現成規則，不改參數）
3. 對每個「截止日當天訊號為 BUY_WATCH/BUY_TRIGGER/HOLD」的
   股票，量測其後 30 天的真實報酬
4. 對照組：同期所有其他有價股票的中位報酬
5. 輸出：訊號組 vs 對照組的報酬差、勝率、逐檔明細

**同時修 D-1**：跨股票橫斷面比較（同一天、不同股票、按分數排序
分五分位），消除時間混淆。輸出「分數五分位 vs 未來報酬」的
橫斷面單調性檢驗。

### R-2 評分權重敏感度掃描（對應 D-3）

在 R-1 的橫斷面框架上，對 8 個評分成分逐一做 leave-one-out
（拿掉該成分重算分數 → 看五分位單調性/報酬差是否改善）。
特別驗證假設：「拿掉 recency 後預測力上升」。
輸出：各成分的邊際貢獻表 + 建議權重組。

### R-3 經理人綜合分析層（對應 D-5）★使用者核心需求

新增 `/api/dossier/<SYM>`：把該股**全部真實證據**整合成
結構化投資檔案：

```json
{
  "symbol": "NVDA",
  "as_of": "2026-07-04",
  "quant": {"score": 83, "source": "quant", "components": {...}},
  "technicals": {"trend": "below EMA50", "rsi": 41.2, "atr_pct": 3.6},
  "signal": {"state": "EXIT_ALERT", "key_conditions": [...]},
  "sentiment": {"stocktwits_bull_ratio": 0.67, "sample": 18},
  "scorecard": {"final_score": 69.8, "verdict": "...", "top_factors": [...]},
  "evidence": [top 3 真實推文/新聞摘錄],
  "manager_view": {
    "thesis": "一段話論述（由 Gemini 基於上述真實數據生成，嚴禁引入外部虛構事實）",
    "bull_case": "...", "bear_case": "...",
    "conviction": "LOW|MEDIUM|HIGH",
    "recommendation": "AVOID|WATCH|ACCUMULATE|HOLD|REDUCE",
    "position_guidance": "基於 ATR 的實際可執行倉位建議（錨定現價）"
  },
  "reliability_note": "backtest 樣本外驗證狀態的誠實聲明"
}
```

manager_view 的 AI 生成部分必須：(a) 只引用 payload 內的真實
數據；(b) 附 reliability_note 誠實聲明目前驗證狀態；(c) AI 失敗
時優雅降級為純數據 dossier（不擋核心功能）。

### R-4 Position sizing 錨定現價（對應 D-6）

`entry` 改為最新收盤價；EMA20 改為「參考位」欄位。進場區 =
現價 ±0.5 ATR。同步修 `/api/signal` 與前端顯示。

### R-5 訊號快照落地（對應 D-7）

新表 `signal_history(symbol, date, signal, score, close, ...)`。
每日背景排程對所有有價股票落一次快照（冪等，同日重跑覆蓋）。
這是未來計算真實命中率的地基。
